 prepare une interface de connexion par email et otp, et une recherche multiselect, je te mettrai à disposition les api d'ici là:

1. Authentification / Connexion
Base URL : ${NEXT_PUBLIC_API_BASE_URL}/api/v1

Méthode	Endpoint	Description
POST	/api/v1/auth/login	Login (étape 1 - envoi OTP)
POST	/api/v1/auth/confirm-login	Confirmation OTP (étape 2)
GET	/api/v1/auth/me	Profil utilisateur connecté
POST	/api/v1/admin-auth/password-reset/request	Demande de reset mot de passe
POST	/api/v1/admin-auth/password-reset/reset	Reset mot de passe avec token
Login (étape 1) - Body :


{ "email": "string", "backend_url": "string (optionnel)" }
Confirm Login (étape 2) - Body :


{ "email": "string", "code": "string (code OTP 6 chiffres)" }
Header d'authentification : Authorization: Bearer <token>
Le token est stocké dans localStorage.fanaf_token et le cookie fanaf_token.

2. Recherche de Participants
Méthode	Endpoint	Description
GET	/api/v1/admin/participants	Liste paginée avec filtres
GET	/api/v1/admin/participants/{id}	Détail d'un participant
GET	/api/v1/admin/participants/{id}/invitation	Télécharger invitation (PDF)
GET	/api/v1/admin/participants/speakers	Liste des speakers
Query params pour la recherche :

q - Recherche textuelle
page, per_page - Pagination
type - Filtre type (membre, non-membre, vip, speaker, referent)
organization - ID organisation
country - Code pays
status - Statut (completed, pending)
has_badge, has_invitation, has_piece_photo, has_invoice - Filtres booléens
check_in - Statut check-in
mode_paiement, canal_encaissement - Paiement
from_date, to_date - Plage de dates
exclude_vip, payment_completed - Autres filtres
